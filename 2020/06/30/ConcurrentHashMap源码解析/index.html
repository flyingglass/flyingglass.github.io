ConcurrentHashMap源码解析

2020-06-30
Java

ConcurrentHashMap为Java中常用的并发容器，用于解决HashTable锁住整个hash表的问题，JDK8针对ConcurrentHashMap作了改进和优化，摒弃JDK7的分段锁机制，采用Node + CAS + synchronized保证并发安全。

    2. 插入的键值对hashcode无冲突，采用cas无锁插入
    3. hash表正在扩容，帮忙扩容
    4. 插入链表，采用synchronized关键字锁住尾部节点，如果链表长度大于MIN_TREEIFY_CAPACITY（默认值为8），转化为红黑树，否则直接插入
    5. 检查是否需要扩容

总结

  1. ConcurrentHashMap不允许NULL键，NULL值
  2. ConcurrentHashMap在JDK1.8中采用CAS + synchronized保证并发一致性，区别于jdk1.7中采用ReentrantLock + Segment的方式
  3. ConcurrentHashMap数据结构和HashMap并无本质区别
